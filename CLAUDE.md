## 文档更新规则
- 我让你更新某份文档时，直接修改对应 MD 文件
- 功能确认清单状态变更：我说"完成XX功能"，自动把对应行的 ⬜ 改为 ✅，填上完成日期，迭代次数+1
- 设计文档修改后，在文档末尾的修订历史里追加一条记录

## 工作留痕规则
- 按 Phase 拆分日志文件：`docs/logs/Phase_N_日志.md`，每个 Phase 一个文件，按时间倒序排列，新会话追加在文件顶部
- 每次会话结束后，追加一节：`## Session #N — YYYY-MM-DD`，包含：完成事项、遇到的错误及修复、架构调整、遗留问题、下个会话计划
- Phase 结束后，在对应日志文件末尾追加一节：`# Phase N 汇总`，汇总该 Phase 所有会话
- **写入方式**：先 Write 新内容到临时文件（如 `docs/logs/_new_entry.md`），再 Bash `cat` 拼接到旧文件前面，最后删掉临时文件。全程不 Read 旧日志，避免 token 膨胀
